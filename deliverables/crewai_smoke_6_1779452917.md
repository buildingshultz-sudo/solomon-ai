# API Signup Guide for the YouTube Data API (Free Tier)

## Executive Summary
This document provides a complete, end-to-end process for registering and activating the YouTube Data API (v3) on the free tier within Google Cloud Platform (GCP). It is written for builders who need reproducible steps, security guardrails, and an operational playbook to obtain production-ready credentials. Key outcomes include: (1) a properly configured Google Cloud project with the YouTube Data API enabled, (2) scoped credentials for server-side, client-side, or installed app use cases, (3) quota management practices aligned with the default 10,000-unit daily budget, and (4) a day-by-day execution timeline to move from zero to tested API calls in under one week. The guide emphasizes minimal manual intervention, automation-readiness, and documentation standards suitable for Solomon’s Forge deliverables.

## Introduction

The YouTube Data API (v3) is Google’s RESTful interface for accessing YouTube resources—channels, videos, playlists, comments, and more. For Solomon’s Forge and the Building Shultz ecosystem, the API is foundational for several initiatives:

- **IronEdit**: ingesting channel metadata, captions, and analytics automatically to accelerate editing workflows.  
- **Content Intelligence**: producing topic research, keyword metadata, and performance dashboards inside internal tools.  
- **Builders AI Blueprint**: referencing real API-driven examples to educate tradespeople on AI-assisted content creation.

Google provides a generous free tier—10,000 quota units per day—with optional paid extensions when scaling beyond the default limits. Unlike consumer-facing OAuth flows, the API onboarding process requires deliberate configuration inside Google Cloud Console. The objective of this manual is to eliminate guesswork, formalize guardrails, and ensure the credentials can move from dev to prod with minimal rework.

## Step-by-Step Signup Procedure

### Step 1: Access Google Cloud Console
1. Navigate to [https://console.cloud.google.com/](https://console.cloud.google.com/).  
2. Log in using the designated Google account. Confirm 2FA challenges immediately to avoid session timeouts.  
3. Once inside, the console will display either a project dashboard (if existing projects) or prompt to create a new one.

### Step 2: Create a Dedicated Project
1. Click the **project selector** (top-left near the Google Cloud logo).  
2. Choose **New Project**.  
3. Provide:
   - **Project name:** `SolomonsForge-YouTube-API` (or similar naming standard: `<Org>-<Product>-<Env>`).  
   - **Organization:** Select the workspace if available; otherwise it defaults to “No organization.”  
   - **Location (Folder):** Optional; useful for grouping projects by product line.  
4. Click **Create**. Wait for confirmation toast.

**Best Practice:** Keep YouTube API in its own project. This isolates quotas, IAM permissions, and audit logs from other services, reducing blast radius during experimentation.

### Step 3: Link Billing Account (if prompted)
1. From the navigation menu (☰), go to **Billing**.  
2. If no billing account exists, click **Add billing account**, follow the prompts, and verify card details.  
3. Return to the project dashboard, ensure the new project is linked to the billing profile. (Billing → Account Management → select project.)

### Step 4: Enable the YouTube Data API v3
1. In the top search bar, type “YouTube Data API v3.”  
2. Select the official entry published by Google.  
3. Click **Enable**.  
4. Once enabled, the API metrics overview page becomes available.

_Reference: Google Developers documentation confirms YouTube Data API v3 is the current stable release and is enabled per project (developers.google.com/youtube/v3/getting-started)._

### Step 5: Configure OAuth Consent Screen
Even if using API keys for server-side access, Google requires an OAuth consent configuration before issuing OAuth client IDs.

1. Navigate to **APIs & Services → OAuth consent screen**.  
2. Choose **External** (recommended when the app may access data from YouTube accounts beyond the organization).  
3. Populate required fields:
   - **App name:** `Solomon Forge YouTube Connector`  
   - **User support email:** contact@solomonsforge.com  
   - **Developer contact info:** same or additional emails for notifications.  
4. Scope selection: add YouTube scopes only when needed (e.g., `https://www.googleapis.com/auth/youtube.readonly`).  
5. For early-stage use, keep the app in **Testing** mode (limits to 100 test users). Before public release, submit for verification.  
6. Save and continue through Summary.

### Step 6: Create Credentials
Go to **APIs & Services → Credentials → + Create Credentials**. Three common credentials:

1. **API Key (server-to-server, no user data)**
   - Select **API key**.
   - Immediately click **Restrict Key**:
     - **API restrictions:** choose “Restrict key,” then select **YouTube Data API v3**.
     - **Application restrictions:** 
       - For backend usage, choose **IP addresses** → add static egress IP from the VPS.  
       - For browser-based usage, select **HTTP referrers (web sites)** and list allowed domains.  
   - Save. Document the key securely.

2. **OAuth Client ID – Web Application (for hosted dashboards or IronEdit web features)**
   - Choose **OAuth client ID** → Application type: **Web application**.
   - Set **Authorized JavaScript origins** (e.g., `https://app.ironed.it`).
   - Set **Authorized redirect URIs** (e.g., `https://app.ironed.it/oauth2/callback`).
   - Click **Create**. Download JSON credentials; store in encrypted secrets manager.

3. **OAuth Client ID – Desktop Application (for CLI tooling)**
   - Application type: **Desktop**.
   - Use this for admin tests or localized scripts.
   - Download credentials, protect with OS keychain or password manager.

4. (Optional) **Service Account** – Not typical for YouTube Data API because most endpoints require end-user authorization. Only use if interacting with YouTube resources owned by the same Google Workspace that created the service account and explicitly shared channel ownership.

### Step 7: Verify Quota and Metrics
1. Navigate to **APIs & Services → Dashboard → YouTube Data API v3**.  
2. Click **Quotas** tab; verify default allocations:
   - **Queries per day:** 10,000 units (free tier).  
   - Additional limits such as “Queries per 100 seconds” for user/per project may appear.  
3. Note the operations and cost:
   - `search.list` = 100 units per call.  
   - `videos.list` = 1 unit per call (when using `id` parameter) or higher with `chart=mostPopular`.  
   - `playlistItems.list` = 1 unit.  
   (Source: [https://developers.google.com/youtube/v3/getting-started#quota](https://developers.google.com/youtube/v3/getting-started#quota))  
4. Configure alerting (see Section 5).

### Step 8: Store Credentials and Document Access
1. Push keys/secrets into the team’s password manager or secret store (e.g., 1Password, Vault).  
2. Update the SOP repository with:
   - Credential name  
   - Creation date  
   - Owner  
   - Scope/restrictions  
   - Rotation schedule (recommend 90 days for API keys).  

### Step 9: Conduct Initial API Call
Use curl or Postman to confirm functionality:

```bash
curl \
  'https://youtube.googleapis.com/youtube/v3/search?part=snippet&channelId=UCabc123xyz&maxResults=5&order=date&key=YOUR_API_KEY'
```

Successful response returns JSON payload with recent uploads. Log the timestamp, request ID, and confirm latency under 400 ms for baseline metrics.

## Credential Types and Usage Scenarios

| Credential | Use Case | Auth Flow | Security Controls |
|------------|----------|-----------|-------------------|
| **API Key** | Read-only server tasks (listing videos, pulling channel stats). | Simple key in query string. | Restrict by IP and API; rotate quarterly. |
| **OAuth Client (Web)** | Web apps where end users authorize their YouTube accounts to manage content or read private playlists. | OAuth 2.0 Authorization Code Grant with redirect URI. | Enforce HTTPS, store tokens encrypted, implement refresh token rotation. |
| **OAuth Client (Desktop)** | CLI utilities and local admin scripts. | OAuth 2.0 Installed App flow (loopback or out-of-band). | Use OS keychain, limit distribution to trusted engineers. |
| **Service Account** | Rare for YouTube; only when channel explicitly grants access. | JWT assertion to Google OAuth servers. | Share channel ownership with service account, maintain least privilege. |

## Quota Planning and Monitoring

1. **Understand Unit Costs**:  
   - `search.list`: 100 units  
   - `videos.list`: 1 unit  
   - `channels.list`: 1 unit  
   - `playlistItems.list`: 1 unit  
   (Per Google documentation: [https://developers.google.com/youtube/v3/getting-started#quota](https://developers.google.com/youtube/v3/getting-started#quota))  

2. **Daily Budgeting**:  
   - Free tier: 10,000 units/day resets at midnight Pacific Time.  
   - Example budget for Building Shultz dashboards:
     - Daily upload sync (5 `playlistItems.list` calls) → 5 units.  
     - Channel analytics refresh (10 `videos.list` calls) → 10 units.  
     - Trending keyword research (20 `search.list` calls) → 2,000 units.  
     - Total ≈ 2,015 units/day, well below the limit.

3. **Monitoring Setup**:
   - Go to **IAM & Admin → Quotas**; create alert thresholds at 70% and 90%.  
   - Configure email notifications to Jed (owner), Sol (CTO), and ops alias.  
   - Optionally integrate with Cloud Monitoring → create a dashboard widget for API quota usage.

4. **Scaling Beyond Free Tier**:
   - Submit quota increase request directly inside the Quotas page when approaching 80% sustained usage.  
   - Document justification (e.g., “IronEdit beta with 500 testers; expect 250k units/day”).  
   - Google typically reviews within 48 hours.

5. **Cost Control**:
   - Even with higher quotas, there is no direct per-unit billing for YouTube Data API, but Google requires billing-enabled projects for management.  
   - Use Cloud Audit Logs to detect abnormal spikes (possible credential leakage).

## Security, Compliance, and Governance

1. **Least Privilege IAM**:  
   - Assign **Viewer** role for analysts, **Editor** for developers, **Owner** for Sol only.  
   - Avoid sharing personal Google accounts; prefer workspace identities.

2. **Credential Rotation**:  
   - API keys: rotate every 90 days, or immediately if compromise suspected.  
   - OAuth secrets: rotate twice per year.  
   - Document rotation events in the ops log.

3. **Secret Storage**:  
   - Never commit keys to Git.  
   - Use environment variables plus secret managers for runtime access.  
   - For PC Agent actions, ensure scripts pull the key from encrypted storage.

4. **Audit Logging**:  
   - Enable **Cloud Audit Logs** for Admin Read, Data Read, and Data Write activities.  
   - Export logs to BigQuery or Chronicle for long-term retention.

5. **Compliance Considerations**:  
   - If any data from YouTube users is stored (e.g., tokens, profile info), ensure privacy notices are updated.  
   - Follow Google’s API Services User Data Policy—especially restrictions around storing YouTube Analytics data.

## Risk Register and Mitigations

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| Credential leakage (API key exposed). | Unauthorized quota drain or data access. | Medium | Restrict by IP/referrer, rotate, monitor quotas. |
| OAuth consent not verified before public launch. | Users see unverified warning, trust deficit. | Medium | Submit verification once scopes finalized; include privacy policy URL. |
| Quota exhaustion during marketing pushes. | API errors in product dashboards. | Medium | Implement caching, leverage multiple projects, request higher quota early. |
| Billing account suspension (card expiry). | API disabled until resolved. | Low | Enable billing alerts, add backup payment method. |
| Team changes causing undocumented access. | Loss of control, compliance issues. | Medium | Maintain IAM audit log, revoke departing member’s access within 24 hours. |
