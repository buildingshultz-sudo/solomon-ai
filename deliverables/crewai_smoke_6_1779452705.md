  - API key + OAuth client credentials generated, stored securely, and documented.
  - Quota dashboard alerting configured.
  - Verification tests performed (API Explorer + curl sample).

## 3. Google Cloud Project Setup

1. **Sign into Google Cloud Console**  
   - URL: https://console.cloud.google.com  
   - Verify top-right account icon = business account (not personal).

2. **Create Project**  
   - Click project dropdown → “New Project.”  
   - Naming convention: `sf-youtube-integration-prod` (lowercase, hyphen-delimited).  
   - Location: attach to Solomon’s Forge organization/folder if available.

3. **Set Project Defaults**  
   - Navigate to **Project Settings**: add description, associate with relevant labels (e.g., `business=media`, `product=ironedit`).  
   - Record **Project ID** (immutable) and **Project Number** for API console references.

4. **Enable Billing**  
   - Go to **Billing** → link project to existing billing account.  
   - Configure Budget alert: `Budget Name: YouTube API Free Tier`, Amount `$10`, Alerts at 50%, 90%, 100% (purely preventative).

5. **IAM Baseline**  
   - Principle of least privilege:
     - `solomonforge-admin@...` → Owner.
     - `solomonforge-dev@...` → Editor.
     - Service accounts added later for CI/CD.  
   - Turn on Cloud Audit Logs for Data Access (Admin Read, Data Write) to track credential usage.

## 4. Enable the YouTube Data API v3

1. Inside the project, go to **APIs & Services → Library**.  
2. Search “YouTube Data API v3” (from Google).  
3. Click **Enable**.  
4. Post-enable sanity checks:
   - API listed under **Enabled APIs & Services**.
   - Note Terms of Service link; accept if prompted.

**Tip:** For multi-service environments, also enable `YouTube Analytics API` and `YouTube Reporting API` if planning advanced metrics, but each requires separate quota pools.

## 5. Configure OAuth Consent Screen

1. Navigate to **APIs & Services → OAuth consent screen**.  
2. **User Type:** Choose **External** (required if any non-Workspace users authenticate). Internal is allowed only for Google Workspace domains.  
3. **App Registration Fields:**
   - App name: `Solomon Forge YouTube Connector`.
   - User support email: `support@solomonsforge.com`.
   - App logo: 512×512 PNG (optional but professional).
4. **App Domain:**
   - Homepage URL: `https://solomonsforge.com`.
   - Privacy Policy URL: `https://solomonsforge.com/privacy` (temporary stub acceptable but must describe data usage).
   - Terms of Service URL (optional but recommended).
5. **Authorized Domains:** Add `solomonsforge.com`, `building-shultz.com`, `ironedit.app` (future-proof).
6. **Developer Contact Email:** `devops@solomonsforge.com`.
7. **Scopes:** Initially add only the minimum:
   - `.../auth/youtube.readonly`
   - `.../auth/youtube.upload` (if IronEdit will publish videos)
   - `.../auth/youtubepartner` (only if MCN features needed).  
   Use Add or Remove Scopes to keep the list tight.
8. **Test Users:** If app remains in Testing mode (quota limited to 100 refresh tokens), add Jed’s Gmail + Sol’s service email. For production, follow Google verification steps when exceeding 100 users or requesting sensitive scopes.
9. **Submit for Verification?** Not required for private/internal use with under 100 users, but plan ahead: verification can take 2–7 business days if sensitive scopes are declared.

## 6. Create API Credentials

### 6.1 API Key (Server-to-server lightweight access)

1. Go to **APIs & Services → Credentials → + Create Credentials → API key**.  
2. Copy generated key; rename it `ironedit-youtube-data-api-key`.  
3. **Restrict the key** immediately:
   - **Application restrictions:** HTTP referrers (if used in web client) or IP addresses (for server). For backend use, choose **IP address** and list VPS IP `167.99.237.26`.  
   - **API restrictions:** check “Restrict key,” select “YouTube Data API v3.”  
4. Store key in 1Password vault + Google Secret Manager (if using Cloud Functions).  
5. Rotate quarterly or when staff changes.

### 6.2 OAuth 2.0 Client ID (User-auth flows)

1. **Create Credentials → OAuth client ID**.  
2. **Application type:** Choose based on integration:
   - **Desktop app** for PC Agent/CLI tools.
   - **Web application** for IronEdit backend (requires Authorized redirect URIs, e.g., `https://ironedit.app/oauth2/callback`).
3. Provide name `ironedit-web-oauth` or similar.  
4. Add **Authorized redirect URIs** (prod + staging).  
5. Save Client ID and Client Secret.  
6. Download JSON if needed (store encrypted).

### 6.3 Service Accounts (Optional)

- YouTube Data API generally requires user-based OAuth. Service accounts have limited use (only for CMS accounts). Document but do not rely on service accounts unless necessary.

## 7. Secure Credential Storage and Rotation

- **Primary vault:** 1Password Business → shared vault “Platform Secrets.”  
- **Backup:** Encrypted storage on VPS using `sops` + Git repo (private).  
- **Rotation SOP:**
  1. Generate new key.
  2. Update deployment configs (Kubernetes secret, PM2 env file, etc.).
  3. Validate integration.
  4. Delete old key.
- **Monitoring:** Cloud Audit Logs + Security Command Center for key usage anomalies.
- **Principle:** Never embed keys in source control; use environment variables (`YOUTUBE_API_KEY`, `YOUTUBE_OAUTH_CLIENT_ID`, etc.).

## 8. Quota Management and Cost Controls

| Metric | Default Free Tier |
|--------|-------------------|
| Daily quota | 10,000 units per project |
| Cost | $0 (until requesting additional quota) |
| Sample cost | `videos.list` = 1 unit/request; `search.list` = 100 units |

- **Quota Dashboard:** APIs & Services → Dashboards → Quotas → filter YouTube Data API.
- **Alerts:** Use **Quotas → Edit Quotas → Notifications** to set 70% and 90% usage alerts (email + webhook).  
- **Optimization Tips:**
  - Cache responses (Redis/Postgres) to avoid duplicate calls.
  - Batch requests when possible.
  - Avoid `search.list` in high-volume loops; prefer storing video IDs.
  - Use incremental synchronization (store `publishedAt`, `etag` for delta updates).
- **Scaling:** If 10k units insufficient, file quota increase request detailing use case, compliance, and projected load. Expect review within ~48 hours.

## 9. Testing and Verification

### 9.1 API Explorer Smoke Test

1. Go to https://developers.google.com/youtube/v3/docs/videos/list  
2. Click **Try this API**.  
3. Choose `part=snippet,statistics`, `id=<your-video-id>`, select **API key** auth, input the key.  
4. Execute and confirm 200 OK with metadata payload.

### 9.2 curl / HTTP Client Test

```bash
curl \
  -G https://www.googleapis.com/youtube/v3/channels \
  --data-urlencode "part=snippet,statistics" \
  --data-urlencode "mine=true" \
  --data-urlencode "access_token=$OAUTH_ACCESS_TOKEN"
```

- Use OAuth token from `gcloud auth application-default print-access-token` (if using Web OAuth, exchange authorization code via backend).
- Validate JSON structure matches Appendix B schema.

### 9.3 Application Integration Test

- For IronEdit: run staging workflow (fetch channel videos, ensure caching, log quota consumed).
- For Solomon Bot: call API via Node.js client library (`googleapis` npm package v131+).

## 10. Troubleshooting & Support Escalation

| Symptom | Likely Cause | Resolution |
|---------|--------------|-----------|
| `403 quotaExceeded` | Daily 10k units exhausted | Reduce frequency, add caching, request quota uplift. |
| `403 accessNotConfigured` | API not enabled or key restricted to other APIs | Re-check API restrictions on key. |
| `401 invalidCredentials` | Expired OAuth token | Refresh token; ensure offline access requested. |
| Consent screen shows “unverified app” | App still in testing or scopes sensitive | Submit verification or keep to <100 users. |
| `This project cannot use YouTube Data API` | New accounts flagged | Open Google support ticket with business justification and compliance data. |

Support path: Google Cloud Console → Help → Contact support (if paid support). Otherwise use https://issuetracker.google.com/issues?q=componentid:191650.

## Appendix B – API Contract & Data Model

### Key Endpoints Used

| Endpoint | Purpose | Quota Cost | Notes |
|----------|---------|------------|-------|
| `channels.list` | Fetch channel metadata for Building Shultz | 1 unit | Use `mine=true` for authenticated channel. |
| `videos.list` | Retrieve video details (snippet, statistics) | 1 unit | Accepts up to 50 IDs per call. |
| `search.list` | Discover videos by query | 100 units | Expensive; cache aggressively. |
| `playlistItems.list` | Manage uploads playlist | 1 unit | For scheduling and content automation. |

### Sample Data Model (PostgreSQL)

```sql
CREATE TABLE youtube_videos (
    video_id            TEXT PRIMARY KEY,
    channel_id          TEXT NOT NULL,
    title               TEXT NOT NULL,
    description         TEXT,
    published_at        TIMESTAMPTZ,
    tags                TEXT[],
    duration_seconds    INT,
    view_count          BIGINT,
    like_count          BIGINT,
    comment_count       BIGINT,
    etag                TEXT,
    last_synced_at      TIMESTAMPTZ DEFAULT NOW()
);
```

- Store `etag` to detect changes without refetching.
- Index `published_at` for chronological queries.

### OAuth Token Storage

| Field | Description |
|-------|-------------|
| `user_id` | Internal user reference |
| `access_token` | Short-lived token (encrypted) |
| `refresh_token` | Long-lived (encrypted, rotate annually) |
| `scope` | Scopes granted |
| `expires_at` | Expiration timestamp |

Security: encrypt with AES-256 (libsodium) before persisting.

## Appendix C – Compliance & Audit Notes

1. **YouTube API Services Terms of Service:** https://developers.google.com/youtube/terms/api-services-terms-of-service  
   - Prohibit selling access to data, require clear attribution, and demand data deletion upon user request.
2. **Data Handling:** Do not cache personally identifiable information beyond necessity. Provide end-users ability to disconnect (revoke tokens via Google Security settings).
3. **Branding:** If building public-facing app, comply with Google branding guidelines (no implying partnership).
4. **Revocation Testing:** Quarterly, revoke OAuth token from Google Account → ensure application handles 401 gracefully.
5. **Incident Response:** If API key leakage suspected, revoke immediately, rotate, and document incident in security log.
