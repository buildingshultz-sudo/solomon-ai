## 4. Detailed Signup Instructions by Service

### 4.1 YouTube Data API v3 (Google Cloud Console)

#### 4.1.1 Purpose and Usage
YouTube Data API v3 powers content ingestion, analytics, and metadata operations for digital channels operated by Building Shultz (e.g., subscriber analytics, video performance retrieval, playlist management). The free tier typically allows 10,000 quota units per 24 hours by default, sufficient for initial prototypes and dashboards.

#### 4.1.2 Step-by-Step Execution Plan (Chrome Only)
1. **Start Chrome** via PC Agent, ensure no more than two initial tabs.
2. Navigate to [https://console.cloud.google.com/](https://console.cloud.google.com/).
3. Authenticate with the designated Building Shultz Google Workspace account (use the corporate-managed identity; never rely on personal accounts).
4. In the top navigation bar, click the project selector → **“New Project”**.
5. Name the project `BuildingShultz-YouTube-Data` (or align with existing naming conventions). Leave organization set to Building Shultz’s domain.
6. Click **Create**, wait for the success banner, then switch context into the new project via the project selector.
7. From the left sidebar (or search bar), select **APIs & Services → Library**.
8. Search for **“YouTube Data API v3”**, open the tile, click **Enable**. Chrome will open the API detail page; ensure confirmation text “API enabled” appears.
9. Go to **APIs & Services → Credentials**.
10. Press **“+ Create Credentials”** → **API key**. Google Cloud will display a modal containing the generated API key.
11. Copy the key to a secure note within the PC Agent environment. Immediately close the modal by clicking **Close**.
12. Under the key list, click the edit pencil next to the newly created key; apply restrictions:
    - **API restrictions**: set to “YouTube Data API v3.”
    - **Application restrictions**: if unknown, leave as “None” for now, but document that this key is server-side only and will be rotated once final workload is defined.
13. Save changes, confirm the key remains accessible; recopy final key value to avoid caching older strings.
14. Close the tab to comply with the five-tab rule.

#### 4.1.3 Post-Generation Actions
- Paste the recorded key into the `.env` structure as `YOUTUBE_API_KEY=...`
- Document the creation timestamp and the Google Cloud project ID in internal notes.
- Schedule the addition of OAuth 2.0 credentials if the plan includes user-authorized workflows (not required for public data requests but noted for Phase 2).
